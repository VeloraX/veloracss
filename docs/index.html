Documentation

Install the package, learn the rules, and work inside the frozen 0.1 surface.

VeloraCSS ships as compiled CSS, optional runtime helpers, and a manifest-backed API. These docs cover the public rules that should not drift between releases.

Install

npm install veloracss
import "veloracss/css";
import { initVelora } from "veloracss/js";

Rules

Public rules

Prefixes are strict

Selectors use vel-, tokens use --vel-, and runtime hooks use explicit data-vel-* attributes.

Geometry stays sharper

Pill-heavy chrome is not the default. Rounded pills are opt-in rather than the baseline framework look.

The manifest is authority

The full public selector and token surface is generated into dist/velora.manifest.json during build.

Families

Stable 0.1 selector families

Foundation

Layout and composition

Start with the page frame and spacing rhythm, then layer rows, grids, and alignment helpers on top.

vel-shell vel-stack-* vel-grid-* vel-justify-*

Think container, rhythm, layout, and alignment before components.

Tokens and utilities

Spacing, sizing, surfaces, effects

These utilities control spacing, readable widths, surface tone, border treatment, and elevation.

vel-p* vel-gap-* vel-max-w-* vel-bg-* vel-shadow-*

Use these to shape density and hierarchy without custom one-off CSS.

Typography

Readable roles and tone

Type roles cover large hero headings, section titles, body copy, supportive tone, and monospace accents.

vel-display vel-headline vel-body vel-text-* vel-font-*

The goal is consistent roles, not a giant bag of random font overrides.

Core UI

Forms, cards, buttons, alerts

These are the starter controls for real product screens: actions, forms, validation copy, and status messaging.

vel-button* vel-form vel-input* vel-alert*

The baseline stays sharper and more restrained than pill-heavy UI kits.

Navigation and flow

Shell orientation and progress

Use these pieces to orient the page, move between views, and show staged progress without a heavy runtime layer.

vel-navbar* vel-breadcrumb* vel-tabs vel-steps

They are meant for docs shells, dashboards, and setup flows.

Actions and overlays

Focused interaction surfaces

This group covers toolbars, grouped actions, menus, modal shells, toast feedback, and the small runtime hooks that support them.

vel-action-* vel-dropdown* vel-modal* data-vel-toggle

The JS surface stays narrow: open, close, and toggle only.
