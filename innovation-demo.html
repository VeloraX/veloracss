01 — Color Genetics

One number.
An entire color universe.

Tailwind ships 22 hardcoded color palettes — 1,540 swatches you can't change without config. velora.css 
/* The entire palette from one variable */
--vel-dna-hue: 258;

--vel-color-primary:  oklch(65% 0.21 var(--vel-dna-hue));

document.documentElement.style.setProperty('--vel-dna-hue', 145);

03 — CSS State Machine

Tabs, toggles, and state.
Zero JavaScript.

VeloraCSS uses :has() — the "parent selector" — to make containers aware of their children's state. Radio inputs drive the state machine. CSS reads which one is checked and updates the entire UI. No event listeners. No DOM manipulation.

:has() makes parents aware of their children

The CSS :has() pseudo-class is the first true parent selector. It lets a container change its own styles based on what's happening inside it — without JavaScript reading the DOM.

The entire tab system is ~8 lines of CSS

These tabs are powered by a radio group + :has(). The browser tracks which radio is checked. CSS reads that state and updates the UI. Click between tabs — you're executing a state machine with zero JavaScript.

velora.css 
/* Panel visibility — :has() reads radio state */
.vel-tabs:has(#tab-1:checked) #panel-1 { display: block; }
.vel-tabs:has(#tab-2:checked) #panel-2 { display: block; }
.vel-tabs:has(#tab-3:checked) #panel-3 { display: block; }

/* Active tab highlight — parent reacts to child state */
.vel-tabs:has(#tab-1:checked) label[for=tab-1],
.vel-tabs:has(#tab-2:checked) label[for=tab-2] {
  color: var(--vel-color-primary);
  border-bottom-color: var(--vel-color-primary);
}

What you'd write in Tailwind vs. VeloraCSS

Tailwind approach
// JavaScript required
const [tab, setTab] = useState(0)

// Class toggling in JSX
className={tab === 0
  ? 'border-b-2 border-indigo-500'
  : 'text-gray-500'}

// Event handler
onClick={() => setTab(0)}
VeloraCSS approach
<!-- Just HTML + CSS, no framework -->
<input type="radio" id="t1" checked>
<label for="t1">Tab 1</label>

/* In CSS */
.tabs:has(#t1:checked) #panel-1 {
  display: block;
}

The Tailwind pattern requires a JavaScript framework, component state, and event listeners. VeloraCSS requires none of that — it works in plain HTML.

04 — Scope Theming

One attribute.
An entirely different world.

Set data-vel-theme on any element and everything inside automatically inherits a new color universe. The tokens cascade down the DOM tree. No class swapping. No CSS-in-JS. No style recalculation. Pure cascade.

data-vel-theme="default"
◈
Token Cascade
All colors inside this zone derive from the active theme tokens.
⬡
Zero JS
The theme switch is a single attribute change on the container element.
↬
Nestable
Themes can be nested — a child zone can have its own theme inside a parent zone.
⟁
Instant
No animation needed — CSS custom properties transition smoothly by default.
velora.css 
/* Theme definitions — hue shift drives everything */
[data-vel-theme="ocean"]  { --vel-dna-hue: 205; }
[data-vel-theme="forest"] { --vel-dna-hue: 145; }
[data-vel-theme="ember"]  { --vel-dna-hue: 22;  }
[data-vel-theme="aurora"] { --vel-dna-hue: 300; }

/* In JavaScript — the complete theming logic: */
el.dataset.velTheme = 'ocean';
05 — Smart Forms

Form validation that
CSS handles itself.

VeloraCSS uses :has() combined with CSS pseudo-classes like :valid, :invalid, and :placeholder-shown to build a complete validation UI with zero JavaScript. Type in the fields below.

velora.css 
/* Parent reacts to child input validity — :has() + :invalid */
.vel-field:has(.vel-input:invalid:not(:placeholder-shown)) label {
  color: oklch(65% 0.22 25);  /* red-family */
}

.vel-field:has(.vel-input:invalid:not(:placeholder-shown)) .vel-input {
  border-color: oklch(65% 0.22 25);
  box-shadow: 0 0 0 3px oklch(65% 0.22 25 / 0.12);
}

/* Show error message automatically */
.vel-field:has(.vel-input:invalid:not(:placeholder-shown)) .vel-field-error {
  display: block;  /* zero JS needed */
}

/* Valid state — green via oklch */
.vel-field:has(.vel-input:valid:not(:placeholder-shown)) .vel-input {
  border-color: oklch(70% 0.2 162);
}
06 — Fluid Scale

Typography and spacing

velora.css 
/* Fluid type: clamp(min, preferred, max) — no breakpoints */
--vel-text-base: clamp(0.9375rem, 0.9rem + 0.19vw, 1rem);
--vel-text-2xl:  clamp(1.5rem,    1.3rem + 1vw,    2rem);
--vel-text-5xl:  clamp(2.75rem,   2rem   + 3.75vw, 4.5rem);

/* Fluid spacing — same principle */
--vel-space-4: clamp(1rem, 0.8rem + 1vw, 1.5rem);
--vel-space-8: clamp(2rem, 1.6rem + 2vw, 3rem);

/* Usage — just use the token, scaling is automatic */
padding: var(--vel-space-6);    /* fluid 1.5rem → 2.25rem */
font-size: var(--vel-text-xl);  /* fluid 1.25rem → 1.5rem */
